# OpenMC常用编译命令与技巧
OpenMC手册url:https://docs.openmc.org/en/stable/index.html

## 常用源码编译命令（来自官方安装文档摘录）

### 获取源码

`git clone --recurse-submodules https://github.com/openmc-dev/openmc.git`

切换到稳定分支：

`cd openmc`

`git checkout master`

### 典型的 CMake 编译流程（out-of-source）

`mkdir build && cd build`

`cmake ..`

`make`

### Linux/macOS 编译并安装

`mkdir build && cd build`

`cmake ..`

`make`

`make install`

若无权限写入系统目录，可指定安装前缀：

`cmake -DCMAKE_INSTALL_PREFIX=$HOME/.local ..`

### 常用 CMake 选项

启用 Profiling：

`cmake -DOPENMC_ENABLE_PROFILE=on /path/to/openmc`

指定构建类型（优化级别）：

调试模式（相当于 `-O0 -g`）：

`cmake -DCMAKE_BUILD_TYPE=Debug /path/to/openmc`

发布模式（相当于 `-O3 -DNDEBUG`）：

`cmake -DCMAKE_BUILD_TYPE=Release /path/to/openmc`

带调试信息的优化（默认，约等于 `-O2 -g`）：

`cmake -DCMAKE_BUILD_TYPE=RelWithDebInfo /path/to/openmc`

选择特定 HDF5 安装：

`export HDF5_ROOT=/opt/hdf5/1.8.15`

`cmake /path/to/openmc`

或单次命令写法：

`HDF5_ROOT=/opt/hdf5/1.8.15 cmake /path/to/openmc`

### MPI/并行相关示例（来自依赖说明）

编译 OpenMC 时偏好并行 HDF5：

`cmake -DHDF5_PREFER_PARALLEL=on -DOPENMC_USE_MPI=on ..`

使用 DAGMC（需指定安装前缀）：

`cmake -DOPENMC_USE_DAGMC=on -DCMAKE_PREFIX_PATH=/path/to/dagmc/installation ..`

使用 libMesh（通常需 MPI）：

`cmake -DOPENMC_USE_LIBMESH=on -DOPENMC_USE_MPI=on -DCMAKE_PREFIX_PATH=/path/to/libmesh/installation ..`

### Python API 可编辑安装

`python -m pip install -e .[test]`
