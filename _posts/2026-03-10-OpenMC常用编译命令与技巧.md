# OpenMC 常用编译命令与技巧

> OpenMC 官方文档：<https://docs.openmc.org/en/stable/index.html>

## 一、源码编译速查

### 1）获取源码

```bash
git clone --recurse-submodules https://github.com/openmc-dev/openmc.git
cd openmc
git checkout master
```

### 2）标准 CMake 编译流程

```bash
mkdir build && cd build
cmake ..
make
make install
```

### 3）常用 CMake 构建类型

- **Debug**（约等于 `-O0 -g`）

```bash
cmake -DCMAKE_BUILD_TYPE=Debug /path/to/openmc
```

- **Release**（约等于 `-O3 -DNDEBUG`）

```bash
cmake -DCMAKE_BUILD_TYPE=Release /path/to/openmc
```

- **RelWithDebInfo**（常用，约等于 `-O2 -g`）

```bash
cmake -DCMAKE_BUILD_TYPE=RelWithDebInfo /path/to/openmc
```

### 4）MPI / 并行相关示例

- 优先使用并行 HDF5：

```bash
cmake -DHDF5_PREFER_PARALLEL=on -DOPENMC_USE_MPI=on ..
```

- 启用 DAGMC（需指定安装前缀）：

```bash
cmake -DOPENMC_USE_DAGMC=on -DCMAKE_PREFIX_PATH=/path/to/dagmc/installation ..
```

### 5）Python API 可编辑安装

```bash
python -m pip install -e .[test]
```

## 二、Docker 开发环境配置

适用于 OpenMC 官方 Docker 镜像，以及使用 Docker + VS Code（或 Cursor 等衍生编辑器）进行开发的场景。

> 说明：镜像内调试工具通常不完整，建议额外配置语言服务与构建辅助文件。

### 1）C++ 插件（clangd）

生成 `compile_commands.json`：

```bash
cmake -S . -B build -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
```

检查文件是否成功生成（确保 clangd 可正确索引、导航与跳转）：

```bash
ls build/compile_commands.json
```

### 2）VS Code 终端输出保留

增大缓冲区行数操作为：

`Ctrl+,` 打开设置，搜索 `terminal.integrated.scrollback`，默认值 `1000`，改为 `5000`~`10000`。

或在 `settings.json` 中直接添加：

```json
{
  "terminal.integrated.scrollback": 10000
}
```

如需暂停自动滚动以阅读输出，右键终端区域选择 **Toggle Scroll Lock**（快捷键 `Ctrl+Shift+L`）。

## 三、Git 常用操作

### 1）回退到上一个提交（保留修改）

```bash
git reset --soft HEAD~1
```

### 2）强制推送（谨慎使用）

```bash
git push --force-with-lease
```
